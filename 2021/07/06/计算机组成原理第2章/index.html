海明校验码

思路； 分组校验->多个校验位->校验位标注出错位置

1010 -> 1011
校验位 001

信息位 n
校验位 k
总共n + k个位置 1个正确状态 + 2^k 种状态
2^k >= n + k + 1

n 1 2~4 5~11 12~26 27~57 58~120
k 2 3 4 5 6 7

eg 信息位 1010
1.确定海明码的位数 2^k >= n + k + 1
n = 4 -> k = 3
设
信息位D4D3D2D1(1010),共4位,
校验位P3p2p1 ,共3位
对应海明码 H7H6H5H4H3H2H1

2.确定校验位的分布

H7 H6 H5 H4 H3 H2 H1
D4 D3 D2 P3 D1 P2 P1
1 0 1 ? 0 ? ?

校验位pi放在海明位号2^(i-1)的位置上
1 2 4位
信息位按顺序放到其余位置上

3.求校验位的值
H3: 3 -> 011
H5: 5 -> 101
H6: 6 -> 110
H7: 7 -> 111
p3 p2 p1
竖着将所有1的位置进行异或
p1 = H3^H5^H7 = D1^D2^D4 = 0^1^1 = 0
P2 = H3^H6^H7 = D1^D3^D4 = 0^0^1 = 1
p3 = H5^H6^H7 = D2^D3^D4 = 1^0^1 = 0

H7 H6 H5 H4 H3 H2 H1
D4 D3 D2 P3 D1 P2 P1
1 0 1 0 0 1 0

2.定点数的表示和运算

无符号数

整个机器字长全部二进制位均为数值位，没有符号位，相当于数的绝对值。

n位的无符号数表示范围为 0~2^n-1

有符号数

最高位为符号位

真值 机器数
+156D = 0 1001 1100B
-154D = 1 1001 1100B

小数点：隐含存储(定点数：事先约定； 浮点数: 按规则浮动)

定点小数

纯小数 0 - 1

x0(符号位) 小数点位置(隐含) x1 x2 …(数值部分)

+0.75D = 0.11B 存储为011
-0.75D = 1.11B 存储为111

k-1 × 2^-1 + k-2 × 2^-2 + … + k-m × 2^-m

0 ~ 1-2^-2

-(1-2^(-n)) ~ 1-2^(-n)

定点整数

x0(符号位) x1 x2 …(数值部分) 小数点位置(隐含)

表示范围

绝对值 0 ~ 2^n - 1

-(2^n - 1) ~ 2^n - 1

原码表示法

(带符号的绝对值表示)

整数

x = +1110
[x]原 = 0,1110

y = -1110
[y]原 = 2^4 + 1110 = 1,1110

用逗号将符号位和数值部分隔开

小数

x = +0.1101 [x]原 = 0.1101

x = -0.1101 [x]原 = 1 + 0.1101 = 1.1101

x = +0.1000000 [x]原 = 0.1000000

x = -0.1000000 [x]原 = 1 + 0.1000000 = 1.1000000

用小数点将符号位和数值部分分隔开

若字长为n+1, 则原码小数的表示范围为 -(1 - 2^(-n)) ~ 1 - 2^(-n) (关于原点对称)

在计算机中, 与 . 都不需要存储 是人类便于阅读约定的

补码表示法

对于原码
0000 1110
1000 1110
相加
1001 1100

无符号数
14 + 142 = 156 与上面对应
对于有符号数
14 + -14 = 0 不对应

-5 ≡ 7 (mod 12)
一个负数加上模为该负数的补数

-1011 ≡ 0101 (mod 2^4)

正数 补码与原码相同
负数 补码为数值部分按位取反，末尾加1(取反加1)
也可由补码求的原码

整数

eg x1 = +1010 ， x2 = -1010 字长为8位
最高位为符号位
x1补码为 0000 1010
x2原码： 1000 1010
补码： 1111 0101 + 1 = 1111 0110

若字长为n+1, 则补码整数的表示范围为 -2^n ~ 2^n - 1 (比原码多表示-2^n)

小数

补码也为数值部分按位取反，末尾加1(取反加1)

eg x1 = +0.1001, x2 = -0.1001 字长为8位
x1 补码：0.100 1000
x2 原码 1.100 1000
补码 1.011 0111 + 1 = 1.011 1000

移码表示法

对于补码很难判断其真正大小

移码就是在真值x上加上一个常量(偏执值)， 通常这个常数取2^n

eg x1 = +10101 x2 = -10101 字长位8位
x1 移码 = 2^7 + 10101
= 1001 0101
x2 移码 = 2^7 + (-10101)
= 01101011

移位运算

在计算机内没有小数点
所以认为数据相对小数点移动

右移n位：÷ 2^n
左移n位：× 2^n
与加减法结合实现乘除法

算术移位：机器码采用有符号数

符号位不参与移位！！

对于正数 原码、反码、补码都一样 左移右移都补0
对于负数 原码 左移右移都补0
反码 左移右移都补1
补码 左移补0 右移补1

原码算术移位：左移丢1算术出错，右移丢1影响精度

逻辑移位：机器码采用无符号数

逻辑左移时，高位移丢，低位添0
逻辑右移时，低位移丢，高位添0

01010011
逻辑左移：1010 0110
算术左移：0010 0110

循环移位

左移：将最高位放到最低位
右移：将最低位放到最高位

加减运算

补码引入原因： 两个有符号数可以直接相加不需要分类讨论

加减的基本思路
计算[x]补 + [y]补

eg 设机器字长为8位 A = 15 B = -24 求 [A + B]补 [A - B]补

A = +1111 -> 0000 1111(原码) -> 0000 1111(补码)取反加1
B = -11000 -> 1001 1000(原码) -> 1110 1000(补码)取反加1

[A + B]补 = [A]补 + [B]补 = 0000 1111 + 1110 1000 = 1111 0111 -> 1000 1001 (原码)取反加1 真值-9
[A - B]补 = [A]补 + [-B]补 = 0000 1111 + 0001 1000 = 0010 0111 真值39

[-B]补：[B]补 连同符号位一起取反加1

溢出

当超过了机器字长能表示的范围会发生溢出

下溢|负数区–0–正数区|上溢

判断溢出方法

1.采用一位符号位

设A的符号为 As B的符号为Bs 运算结果的符号为Ss 溢出表达式为

AB表示与
A+B表示或
A 表示非

V = AsBs Ss + As Bs Ss

V = 0 无溢出
V = 1 有溢出

2.采用一位符号位，根据数据进位情况判断溢出

        符号位进位Cs    最高位进位C1

上溢 0 1
下溢 1 0

Cs ⊕ C1 = 1 则为溢出

3.采用双符号位
正数符号为00 负数符号为11

01 上溢
10 下溢

3.浮点数的表示和运算

浮点数的表示

定点数：纯小数与纯整数

浮点数：以适当的形式将比例因子表示在数据中，让小数点的位置根据需要而浮动

浮点数的真值 N = r^E × M (r通常为2)

阶码 尾数
Jf | J1J2…Jm | Sf | S1S2…Sn
阶符|阶码的数值部分|数符|尾数的数值部分

阶码：常用补码或移码表示 尾数：常用原码或补码表示

eg:阶码、尾数均用补码表示，求a、b的值
a = 0,01;1.1001
b = 0,01;0.01001

a:阶码0,01对应真值+1
尾数1.1001对应真值-0.0111 = -(2^(-2 ) + 2^(-3 ) + 2^(-4 ))
理解为-111右移四位 = - 7/(2^4) = -7/16

a = 2^1 × (-0.0111) = 2^1 × (- 7 / 16) = -7/8

b:阶码0,01对应真值+1
尾数0.01001对应真值+0.01001 = (2^(-2 ) + 2^(-5 ))
理解为-1001 右移5位 = 9/32

b = 2^1 × (+0.01001) = 2^1 × 9/32 = 9/16

规格化：规定尾数的最高位数必须是一个有效值

左规：当浮点数运算的结果为非规格化时要进行规格化处理，将尾数左移一位，阶码-1(基数为2)

右规：当浮点数运算的结果尾数出现溢出(双符号位为01、10)时,将尾数右移一位，阶码+1(基数为2)

eg a = 010;00.1100, b = 010;00.1000, 求a + b
a = 2^2 × 00.1100， b = 2^2 × 00.1000
a + b = 2^2 × 00.1100 + 2^2 × 00.1000
= 2^2 × 01.0100
= 2^3 × 00.1010

规格化浮点数的特点

IEEE 754标志

ms(数符) + E(阶码部分，用移码表示) + M(尾数部分用原码表示)隐藏表示最高位1

如果M 为 xxxx 则表示尾数为 1.xxxx

类型 数符 阶码 尾数数值 总位数 偏置值(16进制) 偏置值(10进制)
短浮点数 1 8 23 32 7FH 127
长浮点数 1 11 52 64 3FFH 1023
临时浮点数 1 15 64 80 3FFFH 16383

记住阶码其他可推
短浮点数 float
长浮点数 double

规格化的短浮点数的真值为:(-1)^s×1.M×2^(E-127)
规格化的长浮点数的真值为:(-1)^s×1.M×2^(E-1023)

一些规定：
1.E = 0, M = 0 则真值为0
2.E = 0, M != 0 为非规格化数，真值 = (-1)^s×0.M×2^(-126)
3.1 <= E <= 254 真值 = (-1)^s×1.M×2^(E-127)
4.E = 255, M != 0 真值为 NAN
5.E = 255, M = 0 真值为正负无穷(看符号位)

加法器

串行加法器

只有一个全加器，数据逐位串行送人加法器中进行运算。进位触发器用来寄存进位信号，以便下一次运算

如果操作数长为n位，加法就要分n次进行，每一次产生一位和，并且串行逐位地送回寄存器

并行加法器

串行进位的并行加法器：把n个全加器串接起来，就可以进行两个n位数的相加

串行进位又称为行波进位，每一级进位直接依赖于前一级的进位，即进位信号是逐级形成的

并行进位的并行加法器：各级信号同时形成，又称为先行进位，同时进位

单级先行进位方式：组内并行、组间串行的进位方式
