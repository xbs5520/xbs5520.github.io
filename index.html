第十一章 并发编程

2021-06-01

12.1基于进程的并发编程

构造并发程序的最简单方法就是使用进程，在父进程中接受客户端的连接请求，创建一个子进程为每个新客户进行服务

父子进程中的已连接描述符都指向同一个文件表表项，所以父进程关闭它的已连接描述符副本是十分关键的
否则，将永不会释放已连接描述符的文件条目

基于进程的并发服务器

通常服务器会运行很长时间，所以我们必须要包括一个SIGCHLD处理程序，来回收僵死子进程的资源
父进程必须关闭它们各自的connfd副本，必须关闭它的已连接描述符，以避免内存泄漏
直到父子进程的connfd都关闭了到客户端的连接才会终止

进程的优劣

父子进程见共享状态信息，有一个十分清晰的模型：共享文件表，但是不共享用户地址空间

进程有独立的地址空间即使优点也是缺点
优：一个进程不可能不小心覆盖到另一个进程的虚拟内存
缺：独立的地址空间使得进程共享状态信息变得更加困难

12.2基于IO多路复用的并发编程

基于IO多路复用的并发事件驱动服务器

IO多路复用可以用做并发事件驱动程序的基础，在事件驱动程序中，某些事件会导致流向前推进。
一般的思路是将逻辑流模型化为状态机
一个状态机就是一组状态、输出事件和转移，其中转移是将状态和输入事件映射到状态
自循环是同一输入和输出状态之间的转移

IO多路复用技术的优劣

事件驱动设计
优：比基于进程的设计给了程序员更多的对程序行为的控制。
一个基于IO多路复用的事件驱动服务器是运行在单一进程上下文中，
因此每个逻辑流都能访问该进程的全部地址空间，这使得在流之间共享数据变得容易
缺：编码复杂

12.3基于线程的并发编程

线程就是运行在进程上下文中的逻辑流，线程由内核自动调度，每个线程都有自己的上下文，
包括线程ID、栈、栈指针、程序计数器、通用目的寄存器和条件码

基于线程的逻辑流结合了基于进程和IO多路复用流的特性

Posix线程

是C程序中处理线程的一个标准接口定义大约60个函数
运行程序创建、杀死和回收线程，与对线程安全地共享数据，还可以通知对等线程系统状态的变化。

创建线程

int pthread_create(pthread_create *tid, pthread_attr_t *attr, func *f, void *arg)
创建线程

pthread_t pthread_self(void)_
获得自己线程ID

终止线程

当顶层线程例程返回时，线程隐式地终止
显式终止
void pthread_exit(void *thread_return);

回收已终止线程地资源

int pthread_join(pthread_t tid, void **thread_return)

分离线程

在任何时间点上，线程是可结合的或是分离地，一个可结合地线程能被其他线程回收和杀死。
再被其他线程回收之前，它地内存资源是不释放的。
一个分离的线程是不能被其他线程回收或杀死的，它的内存资源在它终止时由系统自动释放

int pthread_detach(pthread_t tid);

初始化线程

int pthread_once(pthread_once_t *once_control, void (*int_routine)(void));

12.4多线程程序中的共享变量

线程很有吸引力的一方面为多个线程很容易共享相同的程序变量。

线程内存模型

每个线程都有独立的线程上下文，包括线程ID、栈、栈指针、程序计数器、通用目的寄存器值和条件码

让一个线程去读另一个线程的寄存器是不可能的，任何线程都可以访问共享虚拟内存的任意位置。
如果某个线程修改了一个内存位置，那么其他每个线程最终都能在它读这个位置时发现这个变化
因此寄存器是从不共享的，而虚拟内存总是共享的

将变量映射到内存

全局变量：定义在函数之外
本地自动变量：定义在函数内部但没有static属性的变量
本地静态变量：定义在函数内部并有static属性的变量

12.5用信号量同步线程

进度图

进度条将n个并发线程的执行模型化为一条n维笛卡尔空间中的轨迹线，每条轴k对应于线程k的进度

信号量

信号量是具有非负整数值的全局变量，只能通过操作P V进行处理
P，V的定义确保了一个正在运行的程序绝不可能进入到这样一种状态，也就是正确初始化了信号量有一个负值
这个属性称为信号不变性

使用信号量来实现互斥

信号量提供了一种很方便的方法来确保对共享变量的互斥访问，基本思想是将每个共享变量与一个信号量s
联系起来，然后P和V操作将相应的临界区包围起来
被保护的信号量叫做二元信号量

12.7其他并发问题

必须小心的编写那些具有称为线程安全性属性的函数，一个函数被称为线程安全的，当且仅当
多个并发线程反复地调用时会产生正确地结果
第一类：不保护共享变量地函数
第二类：保持跨越多个调用状态地函数
第三类：返回指向静态变量地指针的函数
第四类：调用线程不安全函数的函数

可重入性

当函数被多个线程调用时不会引用任何共享数据
线程安全函数包含可重入函数

在线程化的程序中使用已存在的库函数

竞争

当一个程序的正确性依赖于一个线程要在另一个线程到达y点之前到达它的控制流中的x点时，
就会发生竞争。通常发生竞争是因为程序员假定线程将按照某种特殊的轨迹线穿过执行状态空间，
忘记另一条准则：多线程的程序必须对任何可行的轨迹线都正确工作

死锁

信号量引入了一种潜在的令人厌恶的运行时错误，叫做死锁，它指的是一组线程被阻塞了
等待一个永远也不会为真的条件
程序员使用PV操作顺序不当以至于两个信号量的禁止区域重叠
重叠的禁止区域引起了一组称为死锁区域的状态
死锁不总是可以预测的

互斥锁加锁顺序规则：给定所有互斥操作的一个全序，如果每个线程都是以一种顺序来获得互斥锁
并以相反的顺序释放，那么这个程序就是无死锁的。

第十一章 网络编程

2021-05-27

11.1 客户端-服务器编程模型

每个网络应用都是基于客户端-服务器模型的
一个应用是有一个服务器进程和一个或多个客户端进程组成的，服务器管理某种资源，
并且通过操作这种资源来为它的客户端提供服务

一个客户端-服务器事物由四步组成
1.当一个客户端需要服务时，它向服务器发送一个请求，发起一个事物
2.服务器收到请求后，解释它，并以适当的方式操作他的资源
3.服务器给客户端发送一个响应，并等待下一个请求
4.客户端收到相应并处理它

11.2 网络

物理上而言网络是一个按照地理 远近组成的层次系统，最低级为LAN 局域网
最流行的局域网技术是以太网
以太网段包括线缆和一个集线器的盒子构成
使用电缆和网桥，多个以太网段可以连接成较大的局域网，称为桥接以太网

在每台主机和路由器上的协议技术，消除了不同网络之间的差异
这种协议必须提供两种基本能力，命名机制和传送机制

11.3 全球IP因特网

全球IP因特网是最著名和最成功的互联网络实现，每台因特网主机都运行实现TCP/IP协议的软件，
几乎所有现代计算机系统都支持这个协议，英特网的客户端和服务器混合使用套接字接口函数和IO函数来进行通信

TCP/IP协议实际是一个协议族，其中内一个都提供不同的功能

IP地址

一个IP地址就是一个32位的无符号整数，IP地址通常是以一种称为点分十进制表示法来表示的

因特网命名

因特网定义了一组更加人性化的域名，以及一种将域名映射到IP地址的机制
域名集合形成了一个层次结构，每个域名编码了它在这个层次中的位置

因特网连接

因特网客户端和服务器通过在连接上发送和接受字节流来通信。
从连接一对进程的意义上而言，连接是点对点的，从数据可以同时双向流动的角度来说它是双工的
并且从由源进程发出的字节流最终被目的进程以发出的顺序收到它的角度来说它也是可靠的

11.4 套接字接口

套接字接口（socket interface）是一组函数，他们与Unix IO函数结合起来用于创建网络应用

套接字地址结构

从linux内核的角度来看，一个套接字就是通信的一个端点，从Linux程序的角度来看，
套接字就是一个有相应描述符的打开文件

socket函数

int socket(int domain, int type, int protocol);
创建一个套接字描述符，返回的描述符仅是部分打开的，还不能用于读写

connect函数

int connect(int clinetfd, const struck sockaddr *addr, socklen_t addrlen);
建立和服务器的连接
函数试图与套接字地址为addr的服务器建立一个因特网连接，其中addrlen为sizeof
connect函数会阻塞，一直到连接成功建立或是发生错误

bind函数

int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
函数告诉内核将addr中的服务器套接字地址和套接字描述符sockfd联系起来

listen函数

int listen(int sockfd, int backlog);
客户端是发起连接请求的主动实体买服务器是等待来自客户端的连接请求的被动实体。

accept函数

int accept(int listenfd, struct sockaddr *addr, int *addrlen);
函数等待来自客户端的连接请求到达侦听描述符listenfd 然后在addr填写客户端的套接字地址
并且返回一个已连接描述符，这个描述符可被用来利用UnixIO函数与客户端通信

主机和服务的转换

Linux提供了一些强大的函数，实现二进制套接字地址结构和主机名、主机地址、服务器名
和端口号的字符串表示之间的相互转化

1.getaddrinfo 函数
int getaddrinfo(const char *host, const char *service,
const struct addrinfo *hints, struct addrinfo **result);
将主机名、主机地址、服务器名和端口号的字符串表示转化成套接字地址结构

函数默认返回IPv4和IPv6套接字地址
对于host关联的每个地址，函数最多返回三个addrinfo结构

2.getnameinfo函数
int getnameinfo(const struct sockaddr *sa, socklen salen,
char *host, size_t hostlen, char *service, size_t servlen, int flags);

套接字接口的辅助函数

1.open_clientfd
int open_clientfd(char *hostname, char *port)
函数建立与服务器的连接，该服务器运行在主机hostname上，并且在端号口port上监听连接请求

2.open_listenfd
int open_listenfd(char *port);
函数打开和返回一个监听描述符，这个描述符准备号在端口port上接受连接请求

11.5 Web服务器

博弈论

公平组合游戏ICG
若一个游戏满足：

1.由两名玩家交替行动；
2.在游戏进程的任意时刻，可以执行的合法行动与轮到哪名玩家无关；
3.不能行动的玩家判负；
则称该游戏为一个公平组合游戏。
NIM博弈属于公平组合游戏，但城建的棋类游戏，比如围棋，就不是公平组合游戏。
因为围棋交战双方分别只能落黑子和白子，胜负判定也比较复杂，不满足条件2和条件3。

Nim游戏

给定N堆物品，第i堆物品有Ai个。两名玩家轮流行动，每次可以任选一堆，取走任意多个物品，可把一堆取光，但不能不取。取走最后一件物品者获胜。两人都采取最优策略，问先手是否必胜。

我们把这种游戏称为NIM博弈。把游戏过程中面临的状态称为局面。
整局游戏第一个行动的称为先手，第二个行动的称为后手。
若在某一局面下无论采取何种行动，都会输掉游戏，则称该局面必败。
所谓采取最优策略是指，若在某一局面下存在某种行动，使得行动后对面面临必败局面，则优先采取该行动。
同时，这样的局面被称为必胜。我们讨论的博弈问题一般都只考虑理想情况，
即两人均无失误，都采取最优策略行动时游戏的结果。
NIM博弈不存在平局，只有先手必胜和先手必败两种情况。

定理： NIM博弈先手必胜，当且仅当 A1 ^ A2 ^ … ^ An != 0

a为每一堆石子个数
先手必败状态： 不能操作 不管怎么操作都是必胜状态
a1 ∩ a2 ∩ … ∩ an = 0

异或 ∩

先手必胜状态： 可转变为必败
a1 ∩ a2 ∩ … ∩ an ≠ 0

所以任意时刻先手必胜一定不是0
必败一定是0

所以只需判断初始状态 再根据先后手就可得出结论

SG函数

Mex运算
设S表示一个非负整数集合。定义mex(S)为求出不属于集合S的最小非负整数的运算，即：
mex(S) = min{x}, x属于自然数，且x不属于S

在有向图游戏中，对于每个节点x，设从x出发共有k条有向边，
分别到达节点y1, y2, …, yk，定义SG(x)为x的后继节点y1, y2, …, yk
的SG函数值构成的集合再执行mex(S)运算的结果，即：
SG(x) = mex({SG(y1), SG(y2), …, SG(yk)})
特别地，整个有向图游戏G的SG函数值被定义为有向图游戏起点s的SG函数值，
即SG(G) = SG(s)。

SG(x) = 0 必败
SG(x) ≠ 0 必胜

如果有n张图 将其异或起来就是结果
