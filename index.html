关键字volatile有什么含义？并举出例子

volatile提醒编译器它后面所定义的变量随时都有可能改变，
因此编译后的程序每次需要存储或读取这个变量的时候，

关键字static的作用是什么

1.在函数体，只会被初始化一次，一个声明为静态的变量在这一函数被调用的过程中维持其值不变
2.在模块内(函数体外),一个被声明为静态的变量可以被模块内所用函数访问,但不能被模块外其他函数访问。
它是一个本地的全局变量(只能被当前文件使用)
3.在模块内，一个被声明的静态函数只可被这一模块内其他函数调用，
这个函数被限制在声明它的模块本地范围内使用(只能被当前文件使用)

C语言中，为什么static变量只初始化一次

对于所有的对象(不仅仅是静态),初始化都只有一次,而由于静态变量在初始化后一直都没有被销毁,都会保存在内存区域中,
所以，不会再次初始化。存放在静态区的变量的声明周期一般比较长,它与整个程序同生共死,所以它只需初始化一次,
而auto变量,即自动变量,由于它存放在栈区,一旦函数调用结束,就会被销毁

extern”C” 的作用是什么

extern “C”的主要作用是为了能够正确实现C++代码调用其他C语言代码，
加上后会指示编译器这部分代码按C语言进行编译，而不是C++的

使用原码时 对进行加减的数无法判断结果的正负

补码表示法

对于原码
0000 1110
1000 1110
相加
1001 1100

无符号数
14 + 142 = 156 与上面对应
对于有符号数
14 + -14 = 0 不对应

-5 ≡ 7 (mod 12)
一个负数加上模为该负数的补数

-1011 ≡ 0101 (mod 2^4)

正数 补码与原码相同
负数 补码为数值部分按位取反，末尾加1(取反加1)
也可由补码求的原码

整数

eg x1 = +1010 ， x2 = -1010 字长为8位
最高位为符号位
x1补码为 0000 1010
x2原码： 1000 1010
补码： 1111 0101 + 1 = 1111 0110

若字长为n+1, 则补码整数的表示范围为 -2^n ~ 2^n - 1 (比原码多表示-2^n)

小数

补码也为数值部分按位取反，末尾加1(取反加1)

eg x1 = +0.1001, x2 = -0.1001 字长为8位
x1 补码：0.100 1000
x2 原码 1.100 1000
补码 1.011 0111 + 1 = 1.011 1000

反码表示法

正数 反码与原码相同
负数 反码为数值部分按位取反
也可由补码求的原码

整数
eg x1 = +1011 ， x2 = -1011 字长为8位

x1反码为 0000 1011
x2原码 1000 1011
反码 1111 0100

小数
eg x1 = +0.0110 ， x2 = -0.0110 字长为8位

x1反码 0.011 0000
x2原码 1.011 0000
反码 1.100 1111

移码表示法

对于补码很难判断其真正大小

移码就是在真值x上加上一个常量(偏执值)， 通常这个常数取2^n

eg x1 = +10101 x2 = -10101 字长位8位
x1 移码 = 2^7 + 10101
= 1001 0101
x2 移码 = 2^7 + (-10101)
= 01101011

移位运算

在计算机内没有小数点
所以认为数据相对小数点移动

右移n位：÷ 2^n
左移n位：× 2^n
与加减法结合实现乘除法

算术移位：机器码采用有符号数

符号位不参与移位！！

对于正数 原码、反码、补码都一样 左移右移都补0
对于负数 原码 左移右移都补0
反码 左移右移都补1
补码 左移补0 右移补1

原码算术移位：左移丢1算术出错，右移丢1影响精度

逻辑移位：机器码采用无符号数

逻辑左移时，高位移丢，低位添0
逻辑右移时，低位移丢，高位添0

01010011
逻辑左移：1010 0110
算术左移：0010 0110

循环移位

左移：将最高位放到最低位
右移：将最低位放到最高位

加减运算

补码引入原因： 两个有符号数可以直接相加不需要分类讨论

加减的基本思路
计算[x]补 + [y]补

eg 设机器字长为8位 A = 15 B = -24 求 [A + B]补 [A - B]补

A = +1111 -> 0000 1111(原码) -> 0000 1111(补码)取反加1
B = -11000 -> 1001 1000(原码) -> 1110 1000(补码)取反加1

[A + B]补 = [A]补 + [B]补 = 0000 1111 + 1110 1000 = 1111 0111 -> 1000 1001 (原码)取反加1 真值-9
[A - B]补 = [A]补 + [-B]补 = 0000 1111 + 0001 1000 = 0010 0111 真值39

[-B]补：[B]补 连同符号位一起取反加1

溢出

当超过了机器字长能表示的范围会发生溢出

下溢|负数区–0–正数区|上溢

判断溢出方法

1.采用一位符号位

设A的符号为 As B的符号为Bs 运算结果的符号为Ss 溢出表达式为

AB表示与
A+B表示或
A 表示非

V = AsBs Ss + As Bs Ss

V = 0 无溢出
V = 1 有溢出

2.采用一位符号位，根据数据进位情况判断溢出

        符号位进位Cs    最高位进位C1

上溢 0 1
下溢 1 0

Cs ⊕ C1 = 1 则为溢出

3.采用双符号位
正数符号为00 负数符号为11

01 上溢
10 下溢
