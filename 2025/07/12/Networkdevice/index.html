OSI Model devices

what is OSI model

The Open Systems Interconnection model.

Layer 1 devices

Analog modem(模拟调制解调器)

  • convert digital(数字) signal(from digital node) to an analog(模拟) signal
  • create connections between network segments via(经过) PSTN(public switched telephone network) using POTS(plain old telephone system)

Hub

  • function as a concentrator/repeater(集中器/中继器) not care where the signal come and go

Layer 2 devices

Switch

  • Utilizes ASIC(application-specific integrated circuit) chip

    ASIC allows the switch to learn when a device is on network and which ports it is connected to via that device’s Layer 2 MAC address.

  • Maybe a few ports, May be many ports.

  • Maybe simple, Maybe highly complex and programmable.

  • only communicates with local network devices.

Wireless access point(WAP)

  • connects(bridges 桥接) wireless network segments with wire network segments.

    most common type : 802.11 wireless network segments with 8023 Ethernet(以太) network segments

  • only communicates with local network devices.

Layer 3 devices

Multilayer switch(MLS)

  • it provides 2, 3 or higher Layer OSI model services.

  • The most common MLS is a Layer 3 switch

    not only utilizes ASIC chip for switching, but ASIC chip also programed to handle routing functions(communicate and pass data to non-local network devices).

  • Maybe a few ports, May be many ports.

  • highly complex and programmable.

  • expensive! not for common user.

Router

  • Most common network device for connecting different networks.

  • The router uses software programming for decision making as compared to the switch’s use of ASIC chip.

    using programming to keep track of different networks and consider the best possible route to reach those networks.

  • communicate with both local and non-local network devices.

Security devices

Firewall

  • be placed on routers or hosts(software base) or can be its own device.

  • functions at multiple layers of OSI(specifically at 2 3 4 7).

  • block packets from entering or leaving the network.

    Via stateless inspection(无状态检测): firewall will examine every packet against a set of rules. Once matches a rule, the rule is enforced and the specified action is taken.

    Via stateful inspection(有状态检测): firewall only examine the state of the connection between networks. Specifically, when a connection is made from an internal network to an external network, the firewall will not examine any packets returning from the external connection. As a general rule external connections are not allowed to be initiated with the internal network

  • first line of defense in protecting the internal network from outside threats

Intrusion detection system (IDS)

  • A passive(被动) system designed to identify when a network breach or attack against the network is occurring.

    Usually designed to inform a network administrator when a breach or attack has occurred through log files, SMS, and/or an email notification.

  • Cannot prevent or stop a breach or attack on its own.

  • It receives a copy of all traffic(流量) and evaluates it against a set of standards.

    Signature based: evaluates network traffic for known malware(恶意) or attack signatures.

    Anomaly based: evaluates network traffic for suspicious(可疑) changes.

    Policy based: evaluates network traffic against a specific declared security policy.

  • May be deployed at the host level

    Host-based intrusion detection system (HIDS).

Intrusion prevention system (IPS).

  • An active(主动) system designed stop a breach or attack from succeeding in damaging the network.

    Usually designed to perform an action or set of actions to stop the malicious activity.

    Will inform a network administrator through the use of log files, SMS, and/or email notification.

  • All traffic on the network segment flows through the IPS to either enter or leave the segment.

    Like the IDS, all traffic is evaluated against a set of standards.

  • The best placement on the network is between a router (with a firewall) and the destination network segment(目的网络段).

  • It is programmed to make an active response to the situation.

    Block the offending(违规) IP address.

    Close down the vulnerable interface.

    Terminate the network session.

    Redirect(转移) the attack.

Virtual private network (VPN)

  • VPN concentrator will allow for many more secure VPN connections to a network.
  • The concentrator will provide proper tunneling and encryption, depending on the type of VPN connection that is allowed.
  • Most concentrators can function at multiple layers (specifically 2 3 7).
  • Outside of Internet transactions(除了互联网交易) (which use SSL VPN connections at Layer 7) ,most concentrators will function at the network layer (Layer 3) , providing IPsec encryption through a secure tunnel.

Optimization(优化) and performance devices

Load balancer

  • Also be called a content switch or content filter.

  • A network appliance that is used to load balance between multiple hosts that contain the same data spreading out(分散) the workload for greater efficiency.

    Commonly used to distribute the requests (workload) to a server farm(群) among the various servers, helping to ensure that no single server gets overloaded.

Proxy server.

  • An appliance that requests resources on behalf of client machines.

  • It is often used to retrieve(检索) resources from outside untrusted networks on behalf of the requesting client.

  • It hides and protects the requesting client.

  • It can also be utilized to filter allowed content.

  • It can increase network performance by caching(缓存) commonly requested Web pages.
